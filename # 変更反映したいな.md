# 変更反映したいな

[新しい問題(機能追加やエラー修正等)に取り組むとき - イシュー / ブランチ](https://www.notion.so/24827b1a951380c48a8ac21d611b3328?pvs=21)

上のページでイシュー・ブランチを作り、そこでいろいろ機能とか実装したと思います。

そしたら、リモート・ローカルリポジトリ（のブランチ）に反映したいですよね。

その時は**コミット**・**プッシュ**をするんですよ。

# 注意点

コミット、プッシュの際には絶対に気を付けてほしいことがあるので以下のページを参照してください。

[変更反映の際の注意点 - コミット / プッシュ](https://www.notion.so/24827b1a95138084ae8bcc992d8fa06b?pvs=21)

# やり方

コミット・プッシュの方法は簡単です。

コマンドでもありますが、VSCode メインで開発すると思うので VSCode で行う簡単な方法を記述します。要望があればコマンドも書きます。

## 1. ファイルを編集する

![image.png](attachment:601d431b-449f-4e41-9143-4dc9ad5fb3ce:image.png)

ファイルを編集するとファイル名の右側に「M」や「U」などの文字が付与され、Git に認識された状態になります。

## 2. 変更したファイルをステージする

![image.png](attachment:cfcf14fb-8d5f-4420-8f9e-83fe8cf2dc5b:image.png)

Git に認識されたファイルは左側メニューの画像では枝みたいなマークを選択すると、変更したファイルが一覧として表示されます。

そしたら、変更かファイル名の右側にプラスマークがあるのでコミットしたいファイルを選んでプラスを押してください。

プラスを押されたファイルは、**ステージ**といってコミット対象のファイルになります。

![image.png](attachment:b33526bb-c579-487b-ba5c-2e1d4c56a08a:image.png)

この状態で**コミット**するのですがその前に**コミットメッセージ**を書きます。

## 3. コミットメッセージを書く

コミットボタンの上にメッセージを入力する欄があります。そこにメッセージを入力した後、コミットを行うことができます。

ですがこの時に書く、コミットメッセージは以下の例のように工夫してほしいです。

> 例 **feat: 投稿データ取得機能を実装 #1**

構造は以下の通りです。

**ルール**は以下のトグルリストを確認してください。

**プレフィックス**: **コミット内容** #**イシュー番号**

- プレフィックス
  - どんなコミットなのかシュッと分かるように Prefix としてコミットの種別を書きます
  - Semantic Commit Message と同様の種別を使います
    - `chore`
      - タスクファイルなどプロダクションに影響のない修正
    - `docs`
      - ドキュメントの更新
    - `feat`
      - ユーザー向けの機能の追加や変更
    - `fix`
      - ユーザー向けの不具合の修正
    - `refactor`
      - リファクタリングを目的とした修正
    - `style`
      - フォーマットなどのスタイルに関する修正
    - `test`
      - テストコードの追加や修正
  [Git のコミットメッセージの書き方（2023 年 ver.）](https://zenn.dev/itosho/articles/git-commit-message-2023)
- コミット内容
  ここは自分の判断で、何を実装したのか、修正したのか、などを長くなってもいいので書いてください。
- イシュー番号
  ここは現在取り組んでいるイシューのタイトルにある#番号を書いてください。
  ![image.png](attachment:be3f0260-16c0-4e40-9dcb-6ebc8275031c:image.png)
  画像の例の場合は「1」がイシュー番号です。

## 4. コミットする

コミットメッセージが書き終わったら、コミットボタンを押してコミットできます。

![image.png](attachment:40fa04f1-4b67-44e3-9668-68c162efc1d8:image.png)

![image.png](attachment:7a9e4426-29e3-4648-ac2e-81c64bd72611:image.png)

ですがここまでだと、ローカルのリポジトリのみの更新となっているため、**プッシュ**をすることでコミットをリモートリポジトリに反映します。

## 5. 変更を同期する（プッシュする）

コミット後、コミットボタンが変更を同期するボタンになっています。

これでリモートリポジトリにプッシュができます。

![image.png](attachment:afa31dba-50c3-4c2f-87b0-e512228929f2:image.png)

GitHub で確認すると以下のように変更が反映されていることが分かります。

![image.png](attachment:d46bd75a-b960-4b10-b035-6d668b6adb24:image.png)

以上の手順で変更を反映することができます。簡単ですね。
